# 🐹 Aegis Pet — AI 협업으로 만든 데스크톱 다마고치

> **한 줄 소개**: 업무시간에 함께하는 데스크톱 오버레이 펫. Claude Code + ChatGPT + Godot을 활용해 **아이디어부터 라이브 배포까지 3일 만에** 완성한 AI 협업 실전 사례.

---

## 📥 다운로드 & 실행 방법

### 최신 버전
👉 **https://github.com/an4216/aegis_pet/releases/latest**

### 설치
1. `aegis-pet-vX.X.X-windows.zip` 다운로드 → 압축 해제
2. `aegis-pet.exe` 더블클릭 (설치·관리자 권한 불필요)
3. SmartScreen 경고 → "추가 정보" → "실행"
4. 화면 하단에 알이 나타나면 클릭해서 부화시키기

### 시스템 요구사항
- Windows 10 / 11 (64비트)
- 별도 프레임워크 설치 불필요

### 자동 업데이트
한 번 설치하면 새 버전 나올 때마다 펫이 알려줍니다 (트레이 클릭 한 번으로 설치). 키우던 펫 데이터는 그대로 이어져요.

### 인간과 AI의 역할 분담

| 담당 | 역할 |
|------|------|
| **사람 (기획자)** | 아이디어 · 방향 결정 · 시각적 피드백 · 진화 조건 설계 · 캐릭터 성격 아이디어 · 사용성 개선 요청 |
| **Claude Code** | 문서화 · 코드 구현 · 디버깅 · 테스트 · 빌드 · GitHub 배포 자동화 · 문제 진단 |
| **ChatGPT (이미지 생성)** | 캐릭터 일러스트 (기본형 10종 + 진화형 10종 = 176개 포즈) |
| **Godot Engine** | 게임 런타임 · 크로스플랫폼 빌드 |

### 왜 AI 협업인가

**전통적 방식으로 만들었다면**:
- 기획서 작성: 1주
- 캐릭터 디자인 (외주): 2~4주, 수백만원
- 프로그래밍 (Windows 오버레이 + 게임 로직): 2~3개월
- 배포·업데이트 시스템: 1~2주
- **합계: 3~4개월, 수천만원**

**실제로 걸린 것**:
- **3일, 실질 작업 시간 20시간 내외, 비용은 Claude/ChatGPT 구독료 수준**

핵심은 "Claude가 혼자 다 만들었다"가 아닙니다. **사람이 방향을 잡고 AI가 실행 속도를 극단적으로 올린 것**이 포인트입니다.

## 🛠️ 어떻게 만들었나 (개발 방법론)

### 1. PDCA 사이클 적용

`Plan → Design → Do → Check → Act` 사이클로 진행:

```
[Plan]  기획서 작성 (요구사항 FR-01~29, 리스크, 성공 기준)
   ↓
[Design] 아키텍처 3안 비교 → 선택, 데이터 모델, FSM 정의
   ↓
[Do]    구현을 4개 모듈로 분할 → 세션별로 진행
   ↓
[Check] 자동 테스트 + 시각 검증 (스크린샷 캡처로 UI 확인)
   ↓
[Act]   피드백 반영 → 재빌드 → 자동 업데이트 배포
```

산출물: `docs/01-plan/features/*.plan.md`, `docs/02-design/features/*.design.md`, `docs/02-design/characters.md`

### 2. 문서를 진실의 원천으로

**핵심 원칙**: "코드보다 문서가 먼저"

- 매 기능마다 요구사항(FR-XX)을 기획서에 먼저 등록
- 캐릭터 설정은 별도 md 파일로 관리 → GPT에 그대로 붙여넣어 이미지 생성
- 설계 문서에 함정(예: "SetWindowRgn 매프레임 갱신 시 흰 줄 아티팩트")까지 기록

**효과**: 며칠 지나 다시 봐도 "왜 이렇게 만들었는지" 즉시 이해. AI가 문맥을 잃지 않음.

### 3. 시각적 피드백 루프

Windows 자체 API로 화면을 캡처해서 AI가 결과를 직접 확인:

```
사용자: "말풍선이 안 보여"
   ↓
Claude: [스크린샷 캡처 → 이미지 분석] "말풍선이 클릭 통과 영역 밖에 있어서 렌더링 잘림"
   ↓
Claude: [코드 수정 → 재빌드 → 재캡처] "이번엔 표시됨 확인"
```

**말로 하는 디버깅**이 아니라 **눈으로 하는 디버깅**. 이게 개발 속도의 핵심이었음.

### 4. 캐릭터 아트 생성 파이프라인

```
1. 캐릭터 설정 md 파일 작성 (성격·소품·팔레트·대사)
2. GPT에 붙여넣기 → 4×2 포즈 시트 1장 생성 (같은 이미지 안에서 일관성 유지)
3. Python 스크립트로 격자 슬라이스 (tools/slice_sheet.py)
4. assets/sprites/chars/{종족}/{포즈}.png 자동 배치
5. Godot이 자동 로드
```

**핵심**: 캐릭터당 8장을 따로 생성하면 매번 다르게 생김. **한 이미지 안**에서 만들어야 일관성 확보.

### 5. 반자동 배포 시스템

```
저에게 "릴리스 올려줘" 한 마디
   ↓
[자동] 버전 번호 업 → 빌드 → zip 생성 → git 커밋 → 태그 푸시 → GitHub Release 발행
   ↓
[자동] 15초 뒤 모든 사용자의 펫이 "새 버전 나왔대!" 알림
   ↓
[사용자] 트레이 클릭 한 번 → 다운로드·재시작 → 완료
```

CI/CD 없이도 API + 스크립트로 완전 자동화됨.

## 💡 배운 것 · 팀에 추천하는 것

### 잘 됐던 점

1. **문서 먼저** — 코드 짜기 전에 기획서·설계서를 쓴 게 방향 흔들림 방지에 결정적
2. **작은 릴리스, 빠른 피드백** — 완벽하게 만들려 하지 말고 알파부터 배포. 자동 업데이트가 있으면 두렵지 않음
3. **시각적 확인 자동화** — "화면 캡처 → 이미지 분석" 루프가 UI 디버깅 속도를 10배 올림
4. **AI에게 함정도 기록시키기** — "이거 시도했는데 안 됨"까지 문서에 남겨야 며칠 뒤 같은 실수 반복 안 함

### 겪었던 문제와 해결

| 문제 | 원인 | 해결 |
|------|------|------|
| 클릭 통과 영역 경계에 흰 줄 번쩍 | Windows SetWindowRgn을 매 프레임 갱신 | 64px 격자 스냅으로 갱신 빈도 최소화 |
| 말풍선이 화면에 안 뜸 | 클릭 통과 영역 밖은 렌더링도 잘림 | 영역에 UI도 포함 |
| exe가 실행 안 됨 | 자동 버전 bump 스크립트가 project.godot 파괴 | git 히스토리에서 복원 + 이후 Edit 도구로만 수정 |
| ChatGPT 캐릭터가 매번 얼굴이 다름 | 이미지 여러 장 따로 생성 | 한 이미지 안에 8칸 시트로 한 번에 |

### 팀에 추천하는 워크플로우

**작은 사이드 프로젝트로 시작하기 좋은 조건**:
- 명확한 사용자(자기 자신 or 가까운 사람)
- 재미있는 문제 (동기 유지)
- 3~5일 안에 알파 배포 가능한 규모

**바로 활용 가능한 템플릿**:
- `docs/01-plan/features/*.plan.md` — 기획서 템플릿
- `docs/02-design/features/*.design.md` — 설계서 템플릿 (아키텍처 3안 비교 포함)
- `tools/slice_sheet.py` — AI 이미지 → 게임 리소스 파이프라인
- `scripts/updater.gd` — GitHub Releases 기반 자동 업데이트
- `.claude/CLAUDE.md` (사용자별) — Claude에게 프로젝트 컨텍스트 주입
