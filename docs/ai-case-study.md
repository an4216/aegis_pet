# 웹 개발자가 처음 해보는 스택으로 데스크톱 다마고치 만들기

원래 웹 개발만 하던 사람이에요. Godot 같은 게임 엔진은 손도 안 대봤고 데스크톱 앱 만들어본 적도 없어요. 그런데 며칠 만에 화면 위에 상주하는 다마고치를 만들어서 팀 배포까지 했어요. 뭔가 특별한 기술이 있어서가 아니라, 요즘 상용으로 나와있는 AI 도구 몇 개 조합만으로 여기까지 왔어요.

이 글은 어떻게 진행했는지, 뭘 배웠는지 공유하려고 써요.


## 만든 것

이름은 aegis-pet 이에요. 다운로드는 아래에서 받으시면 돼요.

https://github.com/an4216/aegis_pet/releases/latest

실행하면 화면 하단에 알이 하나 떠요. 클릭 몇 번 하면 부화하고, 그 뒤로는 화면 하단을 자기 마음대로 돌아다녀요. 배고파지면 우클릭해서 먹이 주고, 응아 싸면 청소해주고, 심심하면 놀아주는 정통 다마고치예요. 방치해도 죽지는 않게 만들어서 업무 스트레스는 없어요.

주요 기능은 이 정도예요.

- 알에서 부화하는 캐릭터 10종 (성격이 다 달라요. 라떼 부장님도 있고 칼퇴 유령도 있어요)
- 화면 아무 데나 클릭하면 그대로 통과돼요. 펫 몸통에만 클릭이 걸려요
- 오늘 할 일 3개, 리마인더, 뽀모도로 25/5분 집중 타이머
- 실제 키보드 두드린 횟수나 마우스 클릭 수, 완료한 할 일 개수 같은 업무 활동이 진화 조건이에요
- 캐릭터별 대사 말풍선 (월요일 오전이면 "월요일 힘든 건 네 잘못 아니야" 이런 식으로 떠요)
- 새 버전 나오면 펫이 직접 알려주고 트레이 클릭 한 번으로 자동 업데이트돼요

지금 제 화면 오른쪽 아래에 상주 중이라 그런지 은근히 정들더라고요.


## 처음 써본 기술들

이 프로젝트에 쓴 것 중 저한테 익숙한 건 하나도 없었어요.

- Godot 4 게임 엔진과 GDScript 언어 (둘 다 처음)
- Windows 투명 창 오버레이, 클릭 통과 처리 (처음)
- 트레이 아이콘, 시작 프로그램 레지스트리 등록 (처음)
- 게임 스프라이트 상태 머신, 애니메이션 시스템 (처음)
- C# 소형 헬퍼 프로세스로 OS 정보 폴링 (처음)

원래는 프레임워크 기반 웹 개발자예요. 위 목록 중에 실무에서 써본 게 하나도 없어요. 그래도 만들었어요.


## 진행 방식

방법은 단순했어요.

첫째, Claude Code한테 기획서를 md 파일로 만들어달라고 했어요. 뭘 만들고 싶은지, 어떤 캐릭터가 나올지, 어떤 조건에서 진화할지 같은 걸 문서로 정리한 거예요. 이게 프로젝트 내내 기준점이 됐어요. 나중에 다른 기능 추가할 때도 이 문서를 계속 업데이트하면서 진행했어요.

둘째, 같은 Claude Code한테 그 문서를 근거로 코드를 짜달라고 했어요. Godot 문법을 모르니까 시키는 대로 지시할 수도 없어서, 그냥 자연어로 "이런 조건일 때 이렇게 동작하면 좋겠어요"라고만 요청했어요.

셋째, 캐릭터 이미지는 그 md 파일을 그대로 ChatGPT에 붙여넣어서 만들었어요. 캐릭터 성격, 색깔, 소품 설정이 이미 문서에 다 있으니까 프롬프트를 새로 쓸 필요가 없었어요. 한 캐릭터당 8가지 표정을 격자 이미지 하나에 담아달라고 하고, 그렇게 나온 이미지를 다시 Claude Code에 주면 격자로 잘라서 게임에 넣어주는 파이썬 스크립트까지 자기가 짜서 처리해줬어요.

넷째, 결과 확인하고 마음에 안 들면 그 자리에서 수정 요청했어요. "말풍선이 너무 크다", "화면에 흰 줄 생긴다", "이 캐릭터 눈 더 크게" 같은 식으로요. 이게 반복되면서 완성도가 올라갔어요.

기술적으로 어려운 부분은 제가 몰라도 됐어요. 예를 들어 투명 창에서 클릭이 통과되게 하는 Windows API 특성이라거나, 그 API 때문에 화면 렌더링에 부작용이 생기는 원리 같은 것요. 제가 "흰 줄이 보인다"고만 말하면 Claude가 알아서 원인 찾아내고, 시도해보고, 안 되면 다른 방법으로 다시 시도했어요.


## 실제 걸린 시간과 결과물

며칠에 걸쳐 실제 작업 시간은 20시간쯤 됐던 것 같아요. 낮에 다른 일 하다가 저녁에 몇 시간씩 붙었어요.

이 시간 안에 나온 결과물은 대충 이 정도예요.

- 커밋 60개 이상
- 팀 배포 릴리스 8번 (v0.1.0부터 계속 개선하면서 실제 팀원 반응 받으며 진행했어요)
- 자동 테스트 45개
- 캐릭터 이미지 176장 (기본형 10종 X 8포즈 + 진화형 10종 X 8포즈 + 알 4프레임)
- 100MB 짜리 단일 exe 파일 (설치 없이 더블클릭으로 실행돼요)
- 자동 업데이트 시스템 (제가 새 버전 올리면 사용자 앱이 알아서 알림 띄워요)
- 기획서, 설계서, 캐릭터 설정, 진화 설계 등 문서 여러 개

전통적으로 이걸 만들려면 아트 외주에만 수백만원, 개발자 붙여서 몇 달은 잡아야 하는 규모예요.


## 배운 것

몇 가지 느낀 점이 있어요.

첫째, AI 도구가 아무리 좋아도 방향 없이 코드부터 쳐달라고 하면 결과가 별로예요. 뭘 만들고 싶은지 md 문서로 미리 정리해두고, 그 문서를 기준으로 대화하면 확실히 좋아져요. 며칠 지나서 다시 봐도 "이걸 왜 이렇게 만들었지" 헷갈리지 않는 것도 이 문서 덕분이었어요.

둘째, 완벽하게 만들고 배포하려 하지 말고 대충 되면 바로 팀에 뿌리고 반응 보면서 고치는 게 훨씬 나아요. 자동 업데이트만 붙여두면 두려울 게 없어요. 실제로 배포한 뒤에 "말풍선이 너무 위에 뜬다"거나 "특정 화면에서 흰 줄이 생긴다" 같은 피드백을 받고 그 자리에서 고쳐서 다시 올렸어요.

셋째, 캐릭터 이미지처럼 시각적인 결과물은 말로 설명하지 말고 화면 캡처를 보여주는 게 훨씬 빨라요. Claude Code는 이미지를 볼 수 있어서, 스크린샷 보내면서 "여기 이상해요"만 해도 알아서 원인 찾고 고쳐줘요. 텍스트로 왕복하는 것보다 열 배는 빨라요.

넷째, 실수도 있었어요. 자동 버전 업 스크립트가 잘못 짜져서 프로젝트 설정 파일을 통째로 날린 적이 있어요. 그래서 두 번의 릴리스가 실행 자체가 안 되는 상태로 나갔어요. git 히스토리에서 복원해서 급하게 핫픽스 올렸어요. 반복 실수를 막으려면 스크립트로 자동화한 부분에 안전장치를 걸어두는 게 좋아요.


## 팀에서 해볼 만한 것

여러분이 평소에 "이런 게 있으면 좋을 텐데" 하고 넘어간 것 있으면, 진짜 한번 만들어볼 만해요. 만들어본 적 없는 기술 스택이어도 상관없고, 완벽하지 않아도 상관없어요. 알파 버전부터 배포하면서 고쳐가면 돼요.

이 프로젝트에서 쓴 것 중 여러분 프로젝트에도 붙일 수 있는 것들이에요.

- 기획서를 먼저 md로 정리하는 습관
- 콘텐츠 설정(캐릭터, 대사 등)을 md로 관리하고 ChatGPT에 그대로 붙여넣기
- 격자 이미지를 게임 리소스로 자르는 파이썬 스크립트 (tools/slice_sheet.py)
- GitHub Releases 기반 자동 업데이트 (scripts/updater.gd)

소스는 다 공개돼 있으니 가져다 쓰셔도 돼요.

https://github.com/an4216/aegis_pet


## 다운로드하고 실행하는 법

위 링크에서 windows.zip 파일 받으시면 돼요. 압축 풀고 aegis-pet.exe 더블클릭이면 끝이에요. 별도 설치 안 해도 돼요.

처음 실행할 때 Windows SmartScreen이 파란 창으로 경고를 띄워요. 서명 안 된 프로그램이라 그런 거니까 "추가 정보" 누르고 "실행"을 누르시면 돼요. 한번 실행하면 이후 새 버전은 펫이 알아서 알려주니 신경 안 쓰셔도 돼요.

키우다 보면 몇 시간 뒤에 진화도 하고, 캐릭터별로 대사도 다르니 뽑기 재미도 있어요. 키워보시고 피드백 주시면 감사하겠습니다.
